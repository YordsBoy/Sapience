label: FHIR-Based Interoperability Engineering
category: Technical
sector: healthcare_social_assistance
horizon: core
cluster_tags:
  - "HL7 FHIR"
  - "Interoperability"
description: Designing, developing, and testing Fast Healthcare Interoperability Resources (FHIR) application‑programming interfaces (APIs) and implementation guides to enable secure, standards‑based data exchange across electronic health‑record systems.
source_frameworks:
  - "HL7 FHIR R5 Standard"
  - "U.S. Office of the National Coordinator (ONC) 21st Century Cures Act Final Rule"
proficiency_levels:
  - level: Awareness
    indicator: Describes resources, profiles, and RESTful operations; explains FHIR resource structure; uses public FHIR sandboxes; understands basic Health Level Seven (HL7) concepts.
  - level: Basic
    indicator: Builds RESTful FHIR endpoints; validates resources with FHIR validator; implements SMART‑on‑FHIR authentication.
  - level: Intermediate
    indicator: Develops custom profiles and extensions; maps legacy HL7 v2 messages to FHIR; implements bulk-data, CDS Hooks, and SMART apps.
  - level: Advanced
    indicator: Optimises API performance; ensures compliance with United States Core Data for Interoperability (USCDI); integrates patient‑generated data from wearables; validates IGs, runs inferno tests, and manages versioning
  - level: Expert
    indicator: Authors national implementation guides; leads health‑information exchange (HIE) architectures; contributes to HL7 Da Vinci and Gravity project standards.
