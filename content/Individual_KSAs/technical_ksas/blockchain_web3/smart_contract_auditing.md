description: Systematic examination of on‑chain smart‑contract code to detect logic flaws, security vulnerabilities, and compliance issues before deployment or during protocol upgrades.
source_frameworks: ["OpenZeppelin Security Handbook","Ethereum Improvement Proposal (EIP)‑1559 Audit Guidelines"]
proficiency_levels:
- level: Awareness
indicator: Reads simple Solidity functions; recognises common re‑entrancy patterns.
- level: Basic
indicator: Uses static‑analysis tools such as Slither; documents integer‑overflow checks.
- level: Intermediate
indicator: Performs manual logic‑flow review; writes unit tests in Hardhat; verifies gas‑optimisation changes.
- level: Advanced
indicator: Conducts formal verification with Coq or Certora Prover; produces public audit reports; recommends upgrade‑safe patterns.
- level: Expert
indicator: Leads multi‑chain audits; develops novel detection heuristics; advises regulators on decentralized‑finance (DeFi) security standards.
